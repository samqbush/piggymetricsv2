# PiggyMetrics — Modernization Plan

> Forward-looking action set. Current-state evidence lives in `ARCHITECTURE.md`
> (cited, not restated here). Generated by the `arch:modernize` skill, 2026-07-05.

## 1. Executive summary

PiggyMetrics is a 9-module Spring Boot / Spring Cloud / Netflix-OSS microservices
demo, frozen on an entirely EOL stack (Java 8, Spring Boot 2.0.3, Spring Cloud
Finchley, Zuul 1, Hystrix, Turbine, `spring-security-oauth2`). The feasibility
spike proved it is **alive-but-crippled**: it compiles on JDK 8 and its unit
tests pass; only the integration tests are red, and solely because one test
dependency (embedded MongoDB) hits a dead download mirror. That makes this a
**freeze-then-lift** job, not a rewrite. The plan: (1) fix the test harness to
get a **green baseline on the current stack**, (2) migrate CI to GitHub Actions,
(3) upgrade the platform to **Java 21 + Spring Boot 3.3 + Spring Cloud 2023.x**,
swapping each dead Netflix component for its modern equivalent (Zuul→Spring Cloud
Gateway, Hystrix→Resilience4j, Ribbon→Spring Cloud LoadBalancer, Sleuth→Micrometer
Tracing), and (4) replace the removed `spring-security-oauth2` stack with **Spring
Authorization Server + OAuth2 Resource Server (JWT)**. The microservice
decomposition stays — demonstrating it is the repo's purpose.

## 2. Current state assessment

See `ARCHITECTURE.md`. One-line recap: everything works on Java 8 today except a
test-harness dependency; every framework and Netflix component in use is EOL; auth
is built on a library that no longer exists in modern Spring Security; CI is dead.

## 3. Feasibility spike result & strategy

  embedded mongo `flapdoodle 1.50.3` → dead mirror (`fastdl.mongodb.org/...i386...` 403).

**Phase 1.** Fixing the embedded-mongo test dependency turns the whole suite green
on the *current* Boot 2.0.3 stack. This is the single most important marker in the
plan: every later upgrade is validated against a green, trustworthy baseline.

**Phase 2** (the first phase that stands up GitHub Actions). CI runs `mvn test` on

*enforced* required status check / branch-protection rule is a manual GitHub UI

*block* merges.

## 4. Target architecture

**Pattern:** keep the **microservices decomposition** (it is the demo's reason to
exist), but **shrink the infra surface** — delete the two Hystrix-only infra
services and fold observability into Actuator + Micrometer + Prometheus/Grafana.

**Recommended stack:**

| Concern | From | To | Framework level |
|---------|------|----|-----------------|
| Runtime | Java 8 | **Java 21 (LTS)** | ⬆️ Upgrade |
| Platform | Spring Boot 2.0.3 | **Spring Boot 3.3.x** | ⬆️ Upgrade (Jakarta namespace) |
| Cloud | Spring Cloud Finchley | **Spring Cloud 2023.0.x (Leyton)** | ⬆️ Upgrade |
| Edge | Netflix Zuul 1 | **Spring Cloud Gateway** | 🔁 Rewrite (no upgrade path) |
| Circuit breaker | Hystrix | **Resilience4j** (Spring Cloud CircuitBreaker) | 🔀 Swap |
| Load balancing | Ribbon | **Spring Cloud LoadBalancer** | 🔀 Swap (built-in) |
| Dashboard/metrics | Hystrix Dashboard + Turbine | **Micrometer + Prometheus + Grafana** (Actuator) | 🗑️ Remove + 🔀 replace |
| Discovery | Eureka | **Eureka** (still supported) | ✅ Keep / ⬆️ upgrade |
| Config | Cloud Config (native) | **Cloud Config (native)** | ✅ Keep / ⬆️ upgrade |
| Tracing | Sleuth | **Micrometer Tracing** (+ OTel/Zipkin bridge) | 🔀 Swap |
| Auth server | `spring-security-oauth2` `@EnableAuthorizationServer` | **Spring Authorization Server** | 🔁 Rewrite (library removed) |
| Resource servers | `@EnableResourceServer` + CustomUserInfoTokenServices | **OAuth2 Resource Server (JWT)** | 🔁 Rewrite |
| Service-to-service auth | `OAuth2FeignRequestInterceptor`/`OAuth2RestTemplate` | **OAuth2 Client** `client_credentials` + Feign/RestClient | 🔀 Swap |
| Token store | `InMemoryTokenStore` (opaque) | **JWT** (stateless, signed) | 🔁 Rewrite |
| Password/secret encoding | `NoOpPasswordEncoder` / `{noop}` | **BCrypt** | ⬆️ Fix |
| HTTP client | OpenFeign | **OpenFeign** (Spring Cloud OpenFeign) | ✅ Keep |
| Test DB | flapdoodle embedded mongo 1.50.3 | **Testcontainers MongoDB** | 🔀 Swap |
| DB image | `mongo:3` | **`mongo:7`** | ⬆️ Upgrade |
| Container base | `java:8-jre` | **`eclipse-temurin:21-jre`** (or Buildpacks/Jib) | ⬆️ Upgrade |
| Coverage | JaCoCo 0.7.6 | **JaCoCo 0.8.12** | ⬆️ Upgrade |
| CI | Travis | **GitHub Actions** | 🔁 Replace |
| Messaging | RabbitMQ (Bus + Hystrix stream) | **RabbitMQ** (Cloud Bus only; Hystrix stream gone) | ✅ Keep, reduced role |

### ADR: Keep microservices; do not collapse to a modular monolith
- **Context:** The system could be simpler as a modular monolith, but its purpose is to demonstrate the microservice pattern.
- **Decision:** Keep the decomposition (Level 1 — upgrade in place). Only remove the Hystrix-only infra services.
- **Alternatives considered:** Modular-monolith rewrite — rejected: defeats the repo's teaching purpose and is a Level-4 rewrite for no functional gain.
- **Consequences:** Retain distributed-systems overhead (discovery, config, gateway) — acceptable and intentional.

### ADR: Freeze-then-lift, not beachhead rewrite
- **Context:** The app compiles and its unit tests pass; only a test dependency is broken.
- **Decision:** Fix the harness to get a green baseline (Phase 1), then upgrade under the net.
- **Alternatives considered:** Beachhead/walking-skeleton on Boot 3 first — rejected: unnecessary when the legacy app is resurrectable at trivial cost.
- **Consequences:** One early phase of test-only changes buys L4 safety for the whole upgrade.

### ADR: Replace Netflix Zuul/Hystrix/Ribbon/Turbine wholesale
- **Context:** All are EOL with no upgrade path on Spring Cloud 2023.x; Zuul 1 and Hystrix are removed from Spring Cloud.
- **Decision:** Rewrite the edge on Spring Cloud Gateway; swap Hystrix→Resilience4j, Ribbon→Spring Cloud LoadBalancer; delete Turbine/Hystrix-Dashboard, replace with Micrometer/Prometheus.
- **Alternatives considered:** Staying on Netflix stack — impossible on the target platform.
- **Consequences:** Gateway routing and resilience config are re-authored; observability shifts from Hystrix streams to metrics scraping.

### ADR: Spring Authorization Server + JWT resource servers
- **Context:** `spring-security-oauth2` (auth server, `@EnableResourceServer`, `OAuth2RestTemplate`) is removed from modern Spring Security.
- **Decision:** Rewrite auth-service on **Spring Authorization Server**; resource servers become **OAuth2 Resource Servers validating JWTs**; service-to-service uses `client_credentials` via OAuth2 Client. Encode secrets/passwords with BCrypt.
- **Alternatives considered:** External IdP (Keycloak/Auth0) — viable and noted as a future option, but a self-contained Authorization Server best preserves the demo's "everything in-repo" nature.
- **Consequences:** Tokens change from opaque (remote check-token) to signed JWT (local validation) — all resource servers must cut over together (Phase 5). Removes the per-request user-info round trip.

## 5. Per-feature migration analysis

Notation: 🟢 lit (already builds/runs). Effort T-shirt in parentheses.

### Account / Statistics / Notification services (data services) — (M each)
1. **Current:** Spring Data Mongo repos, Feign clients, Hystrix fallbacks, OAuth2 resource server. See `ARCHITECTURE.md` §2.
2. **Strategy:** A — Freeze-then-lift → in-place upgrade; Strangler not needed.
3. **Testability:** Milestone **Phase 1** (Testcontainers). Rung **L4**.
4. **Coupling:** Feign contracts between account↔statistics↔auth; all share the OAuth2 token change (Phase 5).
5. **Effort:** M each. **Risk:** Jakarta namespace + Spring Data Mongo query API changes → mitigated by green baseline.
6. **Acceptance:** existing unit + integration suites green on Boot 3 (validated against the seam contracts = REST endpoints in README + Mongo collections).

### Gateway (edge) — (M)
1. **Current:** Zuul 1 prefix routing, serves UI static content (`gateway/`).
2. **Strategy:** A for build, then **rewrite** onto Spring Cloud Gateway (Phase 4).
3. **Testability:** builds Phase 1; routing behavior re-pinned Phase 4. Rung **L3**.
4. **Coupling:** routes to every business service via Eureka; depends on discovery + LB swap.
5. **Effort:** M. **Risk:** routing/aggregation parity; `stripPrefix` semantics differ. Mitigation: characterization tests against the running old gateway (oracle) before cutover.
6. **Acceptance:** each documented route returns byte-parity responses vs. the old gateway for a recorded request set.

### Auth (auth-service + all resource servers) — (L)
1. **Current:** `@EnableAuthorizationServer`, in-memory clients, `InMemoryTokenStore`, `NoOpPasswordEncoder`, remote check-token (`ARCHITECTURE.md` §3).
2. **Strategy:** 🔁 Rewrite onto Spring Authorization Server + JWT.
3. **Testability:** Milestone Phase 1 (unit) → full parity Phase 5. Rung **L4** with named residual risk (token-format cutover).
4. **Coupling:** **highest** — password grant (UI) + client_credentials (services) + every resource server's validation must move together.
5. **Effort:** L. **Risk:** token format change; grant-type deprecation (password grant is removed — UI must move to a supported flow, e.g. authorization_code + PKCE or a BFF token endpoint). Mitigation: single coordinated phase, contract tests on `/oauth2/token` + a protected endpoint.
6. **Acceptance:** UI login + service-to-service calls succeed end-to-end with JWTs; secrets BCrypt-encoded; no `spring-security-oauth2` on the classpath.

### Config / Registry — (S each)
1. **Current:** native Cloud Config server + Eureka.
2. **Strategy:** ⬆️ upgrade in place.
3. **Testability:** Phase 1. Rung L2 (infra shells).
4. **Coupling:** everything boots against them; upgrade first among infra.
5. **Effort:** S. **Risk:** low. 6. **Acceptance:** all services fetch config and register on the new stack.

### Monitoring / Turbine-stream — (XS)
1. **Current:** Hystrix Dashboard + Turbine.
2. **Strategy:** 🗑️ **Remove**; replace with Actuator + Micrometer + Prometheus/Grafana.
3. **Testability:** n/a. 4. **Coupling:** consumes Hystrix streams (deleted with Hystrix).
5. **Effort:** XS (delete + compose changes). 6. **Acceptance:** metrics scrape-able at `/actuator/prometheus`; modules removed from `pom.xml`.

## 6. Phased implementation plan

**Phase-gating rule (regime-aware, applies to every phase):** a phase is not
complete until its Verification & Exit Criteria are executed and recorded. **Lit
phases exit on green CI / runnable commands.** The only pre-testability slice is
the Phase 4 gateway edge rewrite, which exits on characterization-oracle parity +
smoke, not on a not-yet-existing suite. Do not advance until phase N passes.

Because the app is alive from the start, the **first phase establishes the maximum
achievable safety rung (L4) by fixing the test harness** — not by resurrecting a
corpse.

### Phase 3: Platform Upgrade — Java 21 + Spring Boot 3.3 + Spring Cloud 2023.x (T-shirt: L)
**Goal:** Lift every module to the modern platform, swapping in-place-upgradable pieces.
**Regime:** lit. **Safety rung:** L4 (green suite from Phase 1 is the gate). **Prerequisites:** Phase 2.
**Duration:** 2–4 sprints.

#### Tasks
| ID | Task | Component | Blocked by |
|----|------|-----------|------------|
| 3.1 | Bump parent to Boot 3.3.x, `spring-cloud.version` → 2023.0.x, `java.version` → 21 | root `pom.xml` | — |
| 3.2 | Run **OpenRewrite** recipes (`UpgradeSpringBoot_3_3`, Jakarta EE) module by module | all | 3.1 |

#### Risks & Mitigations
- **Risk:** `javax.*`→`jakarta.*` breakage → **Mitigation:** OpenRewrite + green suite.
- **Risk:** Spring Data Mongo query-API changes → **Mitigation:** integration tests from Phase 1.
- **Risk:** auth won't compile (oauth2 removed) → **Mitigation:** **quarantine auth-service in Phase 3**; it is fully rewritten in Phase 5. Keep it on a temporary shim or skip its build until Phase 5 (documented).

#### Decisions made
- **Eureka and native Cloud Config stay** (upgrade-in-place, not replaced).
- Auth-service platform upgrade is **deferred** (not dropped) to Phase 5, where the security rewrite happens together — upgrading it twice is wasteful.
- **Four modules quarantined, not one.** In addition to auth-service, `gateway`
  (Netflix Zuul), `monitoring` (Hystrix Dashboard) and `turbine-stream-service`
  (Turbine Stream) depend on artifacts removed from Spring Cloud 2023.x and are
  excluded from the JDK 21 reactor. gateway/monitoring/turbine → Phase 4; auth → Phase 5.
- **Data-service security → temporary permit-all shim.** The OAuth2 resource-server
  stack (`spring-security-oauth2`) is removed in Spring Security 6; the real JWT
  rewrite is Phase 5, so account/statistics/notification get a permit-all
  `SecurityFilterChain` and their `@PreAuthorize("#oauth2…")` annotations are removed.
- **Config-client bootstrap** kept via `spring-cloud-starter-bootstrap` (lowest-risk;
  preserves the Phase 1 `bootstrap.yml` semantics) rather than migrating to
  `spring.config.import` in this phase.
- **Tests kept on JUnit 4 via `junit-vintage-engine`** (one test migrated to JUnit
  Jupiter because `OutputCapture` was removed) — minimal churn, green baseline preserved.
- **OpenRewrite not used**; the Jakarta / Spring-6 API migration was done manually
  given the small surface (javax→jakarta, `APPLICATION_JSON_UTF8_VALUE`, Jackson 1.x
  →fasterxml, `Assert.hasLength(String)`, `WebSecurityConfigurerAdapter`, Mongo 4
  converters).

### Phase 5: Security Rewrite — Spring Authorization Server + JWT resource servers (T-shirt: L)

**Duration:** 2–3 sprints.

#### Tasks
| ID | Task | Component | Blocked by |
|----|------|-----------|------------|
| 5.1 | Rewrite auth-service on **Spring Authorization Server**; JWT tokens; **BCrypt** secrets/passwords; persistent client + user store | auth-service | — |
| 5.2 | Convert all resource servers to **OAuth2 Resource Server (JWT)**; delete `CustomUserInfoTokenServices` | account/statistics/notification | 5.1 |
| 5.3 | Replace `OAuth2FeignRequestInterceptor`/`OAuth2RestTemplate` with **OAuth2 Client `client_credentials`** | account/statistics | 5.1 |

- Self-contained **Spring Authorization Server** kept in-repo (external IdP like Keycloak = **deferred** future option, not adopted now).

## 8. Migration safety net

- **Oracle & seam contracts:** the **running old system** (published
  `sqshq/piggymetrics-*` images via `docker-compose`) is the behavioral oracle.
  Pin the **seams** — README REST endpoints, MongoDB collections, OAuth2 token
  endpoints, Feign contracts, the external exchange-rates call — as
  golden/characterization snapshots captured in Phase 1. These survive the
  rewrite; unit tests on deleted legacy code do not.
- **Feature flags / parallel run:** the auth cutover (Phase 5) runs new + old
  token validation behind a flag until every service is switched; the gateway
  (Phase 4) can run old Zuul and new SCG side by side and diff responses.
- **Rollback:** per phase = redeploy the previous image (each phase independently
  deployable). Data: Phase 6 mongo upgrade keeps a pre-upgrade dump.
- **Testing strategy:** the green baseline (Phase 1) is carried forward and grown
  on the target stack; Testcontainers replaces embedded mongo; new contract tests
  are added at the auth and gateway seams. Nothing is left permanently quarantined
  except auth-service between Phases 3 and 5 (closed in Phase 5).
- **Observability:** Micrometer + Prometheus + Grafana (Phase 4) prove the new
  system's latency/error/circuit-breaker behavior matches the old Hystrix metrics.
