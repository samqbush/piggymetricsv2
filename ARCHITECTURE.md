# PiggyMetrics — Architecture (Audited Current State)

> Evidence base for `MODERNIZATION_PLAN.md`. Generated by the `arch:document` /
> `arch:modernize` workflow. Every claim below is cited to a file in the repo or
> to a command actually run during the feasibility spike (2026-07-05).

PiggyMetrics is a tutorial **microservices** demo (a financial advisor app) built
on Spring Boot + Spring Cloud + the Netflix OSS stack, orchestrated with Docker
Compose. Nine Maven modules: three business services, six infrastructure
services (`pom.xml` `<modules>`).

## 2. Service / domain map

**Business services** (each owns its own MongoDB database; no shared DB — README "Notes"):

- **account-service** (:6000) — incomes/expenses items, savings, account settings.
  Feign clients to auth & statistics (`account-service/src/main/java/.../client/`).
  Endpoints: `GET/PUT /accounts/current`, `GET /accounts/{name}`, `GET /accounts/demo`, `POST /accounts/` (README).
- **statistics-service** — time-series datapoints, normalizes to base currency via
  external exchange-rates API (`client/ExchangeRatesClient.java`) with a Hystrix
  fallback. Endpoints: `GET /statistics/current`, `/{account}`, `/demo`; `PUT /statistics/{account}`.
- **notification-service** — contact info + notification settings; a `@Scheduled`
  worker sends reminder/backup emails (`service/NotificationServiceImpl.java`,
  `EmailServiceImpl` is `@RefreshScope`). Endpoints: `GET/PUT /notifications/settings/current`.

**Infrastructure services:**

- **config** (:8888) — Spring Cloud Config Server, **native profile**, serving
  `config/src/main/resources/shared/*.yml` over HTTP Basic (`SecurityConfig.java`).
- **registry** (:8761) — Eureka discovery server.
- **gateway** (:4000) — Zuul edge; single entry point, prefix routing, serves the
  UI static content.
- **auth-service** (:5000) — OAuth2 **Authorization Server** (`@EnableAuthorizationServer`).
- **monitoring** (:8080→9000) — Hystrix Dashboard + Turbine aggregator.
- **turbine-stream-service** (:8989) — Turbine Stream over AMQP.

Inter-service calls: Feign + Ribbon + Hystrix, resolved through Eureka. Config
pulled at boot via `bootstrap.yml` (`fail-fast: true`), e.g.
`account-service/src/main/resources/bootstrap.yml`.

## 3. Security architecture (central to modernization)

- **auth-service** issues OAuth2 tokens (`OAuth2AuthorizationConfig.java`):
  `password` grant for the `browser`/`ui` client, `client_credentials` for
  service-to-service (`account-service`, `statistics-service`, `notification-service`).
- **`InMemoryTokenStore`** — tokens are lost on restart; no HA.
- **`NoOpPasswordEncoder` / `{noop}`** client secrets — plaintext (`OAuth2AuthorizationConfig.java`).
- Resource servers use `@EnableResourceServer` + a hand-rolled
  **`CustomUserInfoTokenServices`** that calls `user-info-uri`
  (`http://auth-service:5000/uaa/users/current`, `config/.../shared/application.yml`)
  to validate each token (remote check-token pattern, per-request round trip).
- Service-to-service Feign auth via `OAuth2FeignRequestInterceptor` + `OAuth2RestTemplate`
  (`account-service/.../config/ResourceServerConfig.java`).

**Every class above is built on `spring-security-oauth2`, which is fully removed
from modern Spring Security** — this is the largest single migration surface.

## 4. Deployment & infrastructure

- **Docker Compose** (`docker-compose.yml`, v2.1 format): rabbitmq, config,
  registry, gateway, auth-service, four per-service `*-mongodb` instances (image
  `sqshq/piggymetrics-mongodb` from `mongodb/Dockerfile` = `mongo:3`), the three
  business services, monitoring, turbine. Gateway published on host :80.
- Service images built from prebuilt jars: `Dockerfile` = `FROM java:8-jre; ADD ./target/<svc>.jar`.
- Secrets/passwords injected via env vars (`.env`, `docker-compose` `environment:`).
- `docker-compose.dev.yml` exists for local infra.

## 6. Feasibility spike results (2026-07-05, JDK 8 = zulu 8.0.432)

- `mvn -pl config -am compile` → **exit 0**. Source compiles on JDK 8.
- `mvn -pl account-service -am test` → **14 tests run, 9 errors**. The 5 that pass
  are pure unit tests (service logic, Feign fallback). The 9 failures are **all**
  `Failed to load ApplicationContext` → `de.flapdoodle...DistributionException` →
  `Could not open inputStream for http://fastdl.mongodb.org/osx/mongodb-osx-i386-3.2.2.tgz`
  → **HTTP 403**. The embedded-mongo mirror is retired and the pinned binary is
  **i386** (won't run on Apple-Silicon/arm64 anyway).
- **Interpretation:** the codebase is **not a dead corpse** — it compiles and its
  logic tests pass. The only barrier to a green suite is one **swappable test
  dependency** (embedded MongoDB) hitting a dead mirror + wrong arch. This is a
  classic **freeze-then-lift** candidate: fix the test harness, get green, then
  upgrade under the net.

## 7. Known pain points (deprecated / EOL / risk)

- Entire framework generation EOL (Boot 2.0.3 / Cloud Finchley / Java 8).
- Netflix stack EOL: Zuul 1, Hystrix, Hystrix Dashboard, Turbine, Ribbon.
- `spring-security-oauth2` removed from modern Spring Security.
- Insecure auth defaults: `NoOpPasswordEncoder`, `{noop}` secrets, `InMemoryTokenStore`.
- Test suite red out of the box (dead embedded-mongo mirror + arch mismatch).
- CI is Travis (dead); no GitHub Actions; no lint/format/typecheck gate.
- Deprecated Docker base images (`java:8-jre`, `mongo:3`).
- JaCoCo 0.7.6 cannot instrument Java 9+ bytecode.
